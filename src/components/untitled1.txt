Serangan AP Clone Spam menyiarkan beacon frames dengan SSID access point lain di sekitar, dengan spasi ditambahkan pada nama mereka. Ini menciptakan tampilan beberapa access point dengan nama yang sama. Langkah-langkah serangan:

1. Pindai access point di sekitar dengan perintah `scanap`.
2. Tampilkan daftar access point dengan `list -a`.
3. Pilih access point yang akan dikloning dengan `select -a`.
4. Jalankan serangan dengan `attack -t beacon -a`.

**Proses Kerja List Beacon Spam**

List Beacon Spam adalah metode serangan beacon di mana frame beacon dibuat dari daftar SSID yang diberikan oleh pengguna, kemudian disiarkan ke semua stasiun dalam jangkauan. Berikut langkah-langkahnya:

1. Buat daftar SSID (gunakan salah satu opsi berikut):
   - `ssid -a -g 3` (menghasilkan 3 SSID acak)
   - `ssid -a -n chiken` (menambahkan SSID bernama "chicken")
2. Tampilkan daftar SSID untuk memverifikasi
   - `list -s`
3. Jalankan serangan beacon list spam
   - `attack -t beacon -l`

**Proses Kerja Random Beacon Spam**

Random Beacon Spam menggambarkan serangan di mana pengguna mengirimkan frame beacon dengan SSID acak. Ada dua cara untuk melakukannya dengan Marauder:

1. **Beacon Spam Random List**: Buat daftar SSID acak, kemudian siarkan SSID yang telah dibuat.
   - Contoh: `ssid -a -g 50`, `list -s`, dan `attack -t beacon -l`.
   
2. **Beacon Spam Random Auto**: Setiap kali frame beacon dibuat, SSID acak baru akan dihasilkan dan disiarkan tanpa memerlukan daftar SSID sebelumnya.
   - Contoh: `attack -t beacon -r`.
